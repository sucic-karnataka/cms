JAMstack ಸಾಂಪ್ರದಾಯಿಕ ವೆಬ್ ಅಭಿವೃದ್ಧಿಯ ವಿಧಾನವನ್ನು ಬದಲಾಯಿಸಿದೆ. ಸರ್ವರ್‌ನಲ್ಲಿ ಪ್ರತಿ ಬೇಡಿಕೆಯ ಸಮಯದಲ್ಲಿ ಪುಟಗಳನ್ನು ರಚಿಸುವ ಬದಲು, ಎಲ್ಲವನ್ನೂ ಮುಂಚಿತವಾಗಿ ನಿರ್ಮಿಸಲಾಗುತ್ತದೆ ಮತ್ತು ಸ್ಥಿರ ಫೈಲ್‌ಗಳಾಗಿ ನೀಡಲಾಗುತ್ತದೆ. ಇದರ ಫಲಿತಾಂಶವೆಂದರೆ ವೇಗವಾದ, ಅಗ್ಗವಾದ ಮತ್ತು ಹೆಚ್ಚು ಸುರಕ್ಷಿತ ವೆಬ್.

<!--more-->

## JAM ಎಂದರೇನು?

JAM ಎಂಬುದು **J**avaScript, **A**PIs ಮತ್ತು **M**arkup ಎಂಬ ಪದಗಳ ಸಂಕ್ಷಿಪ್ತ ರೂಪ:

- **JavaScript** — ಕ್ಲೈಂಟ್ ಭಾಗದಲ್ಲಿ ಎಲ್ಲ ಕ್ರಿಯಾತ್ಮಕ ನಡವಳಿಕೆಯನ್ನು ನಿಯಂತ್ರಿಸುತ್ತದೆ.
- **APIs** — ಸರ್ವರ್ ಕಾರ್ಯಗಳನ್ನು ಮರುಬಳಕೆ ಮಾಡಬಹುದಾದ API ಮೂಲಕ ಪ್ರವೇಶಿಸಲಾಗುತ್ತದೆ.
- **Markup** — ನಿರ್ಮಾಣ ಸಮಯದಲ್ಲಿ ಟೆಂಪ್ಲೇಟ್‌ಗಳ ಮೂಲಕ HTML ಅನ್ನು ರಚಿಸಲಾಗುತ್ತದೆ.

## JAMstack ನ ಪ್ರಯೋಜನಗಳು

### ವೇಗ
ಸ್ಥಿರ ಫೈಲ್‌ಗಳನ್ನು CDN ಮೂಲಕ ನೇರವಾಗಿ ಬಳಕೆದಾರರಿಗೆ ತಲುಪಿಸಲಾಗುತ್ತದೆ. ಸರ್ವರ್ ಪ್ರಕ್ರಿಯೆ ಇಲ್ಲದ ಕಾರಣ ಲೋಡಿಂಗ್ ಸಮಯ ಗಣನೀಯವಾಗಿ ಕಡಿಮೆಯಾಗುತ್ತದೆ.

### ಸುರಕ್ಷತೆ
ಸರ್ವರ್ ಡೇಟಾಬೇಸ್ ಇಲ್ಲದ ಕಾರಣ ದಾಳಿಯ ಸಾಧ್ಯತೆ ಕಡಿಮೆ. SQL ಇಂಜೆಕ್ಷನ್ ಅಥವಾ ಸರ್ವರ್ ದಾಳಿಯ ಭಯ ಇರುವುದಿಲ್ಲ.

### ಸ್ಕೇಲೆಬಿಲಿಟಿ
ಹೆಚ್ಚು ಭೇಟಿದಾರರು ಬಂದರೂ ಸ್ಥಿರ ಫೈಲ್‌ಗಳು ಸರಾಗವಾಗಿ ನೀಡಲ್ಪಡುತ್ತವೆ. ಸರ್ವರ್ ಅಧಿಕ ಭಾರ ಹೊರಬೇಕಿಲ್ಲ.

### ಕಡಿಮೆ ವೆಚ್ಚ
ಅನೇಕ JAMstack ಸೈಟ್‌ಗಳನ್ನು ಉಚಿತವಾಗಿ ಅಥವಾ ಅತ್ಯಂತ ಕಡಿಮೆ ವೆಚ್ಚದಲ್ಲಿ ಹೋಸ್ಟ್ ಮಾಡಬಹುದು.

## Hugo ಮತ್ತು JAMstack

Hugo ಒಂದು ಜನಪ್ರಿಯ ಸ್ಥಿರ ಸೈಟ್ ಜೆನರೇಟರ್ ಆಗಿದ್ದು JAMstack ತತ್ವಗಳನ್ನು ಅನುಸರಿಸುತ್ತದೆ. Markdown ಫೈಲ್‌ಗಳಿಂದ ಅತ್ಯಂತ ವೇಗವಾಗಿ HTML ಪುಟಗಳನ್ನು ರಚಿಸುತ್ತದೆ.

```bash
hugo new site ನನ್ನ-ಸೈಟ್
cd ನನ್ನ-ಸೈಟ್
hugo server
```

## ತೀರ್ಮಾನ

JAMstack ಆಧುನಿಕ ವೆಬ್ ಅಭಿವೃದ್ಧಿಯ ಭವಿಷ್ಯ. ವೇಗ, ಸುರಕ್ಷತೆ ಮತ್ತು ಕಡಿಮೆ ವೆಚ್ಚದ ಕಾರಣ ಇದು ಡೆವಲಪರ್‌ಗಳಲ್ಲಿ ಹೆಚ್ಚು ಜನಪ್ರಿಯವಾಗುತ್ತಿದೆ.
